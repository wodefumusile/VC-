# n8n 公众号AI运营 — 部署与配置指南

> 严格对应 DOCX 流程图：14 步完整流水线

---

## 一、快速开始（4 步）

### 1. 启动服务
```bash
cd config
docker compose up -d
```
- n8n: http://localhost:5678
- wordscheck: http://localhost:8080/wordscheck

### 2. 安装社区节点
n8n → Settings → Community Nodes → Install：
- `n8n-nodes-wechat-official`
- `n8n-nodes-wechat-md`

### 3. 配置凭证
n8n → Credentials：
- **WeChat Official Account API**：填入 AppID + AppSecret
- DeepSeek/OpenAI/Claude API Key 直接在 HTTP Request 节点的 Header 里改

### 4. 导入工作流
导入 `workflows/公众号AI运营-完整14步流水线.json`，激活。

测试：
```bash
# 人工审核模式（默认）
curl -X POST http://localhost:5678/webhook/article \
  -H "Content-Type: application/json" \
  -d '{"topic":"AI在制造业的应用"}'

# 自动发布模式
curl -X POST http://localhost:5678/webhook/article \
  -H "Content-Type: application/json" \
  -d '{"topic":"AI在制造业的应用","publishMode":"auto"}'

# 切换模型
curl -X POST http://localhost:5678/webhook/article \
  -H "Content-Type: application/json" \
  -d '{"topic":"AI趋势","model":"claude","publishMode":"manual"}'
```

## 二、14 步流水线对照

| 步骤 | 节点名 | 功能 |
|:---:|------|------|
| ① | 手动输入 / 定时每日9点 | 双触发（Webhook + Schedule） |
| ② | 品牌知识库 | 编辑 brand 对象配置品牌信息 |
| ③ | Prompt增强+模型路由 | 模型配置（deepseek/openai/qwen/claude） |
| ④⑤⑥ | 行业+用户+结构规划 | 一次 AI 调用完成三项 |
| ⑦ | 正文生成 | AI 基于结构写正文 |
| ⑧ | SEO优化 | 独立 AI 调用优化标题/摘要/关键词 |
| ⑨ | 图文排版建议 | AI 推荐 3-5 个配图位置+Unsplash 关键词 |
| ⑩ | AI内容质量检查 | 5维度评分（标题/结构/深度/可读性/合规） |
| ⑪ | 平台违规检测 | wordscheck 敏感词扫描 |
| ⑫ | 生成公众号HTML | WeChat MD 节点格式化 |
| ⑬ | 保存草稿/自动发布 | 双模式 Switch |
| ⑭ | 消息通知 | 4类通知（低质量/违规/草稿/发布成功） |

---

## 三、配置项

### 品牌信息
编辑"② 品牌知识库"节点的 `brand` 对象

### API Key
编辑"③ Prompt增强+模型路由"节点的 `modelConfig`

### 通知 Webhook
环境变量 `NOTIFY_WEBHOOK_URL`（企业微信/飞书/钉钉机器人 Webhook）

### 选题库
编辑"📋 选取选题"节点的 `topics` 数组
