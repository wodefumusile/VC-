# 管理员维护手册

> 适用对象：负责维护系统正常运行的人员（需要一点电脑基础）

---

## 一、查看运行日志

日志文件位置：项目文件夹 → `logs` 文件夹

日志文件按日期命名，例如 `2026-07-13.log`

**用什么软件打开：** 右键 → 打开方式 → **记事本**

**何时需要查看日志：**
- 文章生成失败
- 上传草稿失败
- 系统提示错误

**日志中常见信息：**
- `INFO` — 正常运行记录
- `WARNING` — 警告信息（一般不影响使用）
- `ERROR` — 错误信息（需要关注）

---

## 二、重启服务

### 方法一：正常重启

① 双击 `stop.bat` 关闭系统

② 等待 10 秒

③ 双击 `start.bat` 重新启动

### 方法二：强制重启（系统卡死时）

① 双击 `stop.bat`

② 打开任务管理器（按 `Ctrl + Shift + Esc`）

③ 找到所有 `python.exe` 进程，右键 → **结束任务**

④ 重启 Docker Desktop：
- 右下角托盘区，右键鲸鱼图标 → **Restart Docker**

⑤ 等待 Docker 重新就绪

⑥ 双击 `start.bat`

---

## 三、更新 API Key

适用场景：更换 AI 服务商、Key 过期、余额不足

① 确保系统已关闭（运行 `stop.bat`）

② 在项目文件夹中找到 `.env` 文件

③ 用记事本打开

④ 修改对应配置：

```
# 更换 DeepSeek Key
AI_API_KEY=新的密钥

# 更换为 OpenAI
AI_BASE_URL=https://api.openai.com/v1
AI_MODEL=gpt-4o
```

⑤ 保存文件

⑥ 重新运行 `start.bat`

---

## 四、重新扫码登录

适用场景：微信登录失效、提示需要重新登录

① 确保系统已关闭

② 删除以下文件夹：
```
项目文件夹\data\browser\wechat_clean_profile
```

③ 重新运行 `start.bat`

④ 系统会自动弹出扫码页面

⑤ 用手机微信扫码登录

## 五、备份数据

### 需要备份的内容

| 备份内容 | 位置 | 说明 |
|----------|------|------|
| 配置文件 | `.env` | API Key 等配置 |
| n8n 工作流 | `workflows\` 文件夹 | 工作流定义文件 |

### 备份方法

① 将上述文件和文件夹复制到安全位置（如 U 盘、网盘）

② 建议每周备份一次

③ 更换电脑时，将备份文件复制回新电脑即可恢复

## 六、更新系统版本

收到新版本时：

① 关闭系统（`stop.bat`）

② 备份 `.env` 文件和 `workflows` 文件夹

③ 用新版本替换 `backend` 和 `publish_engine` 文件夹

④ 将备份的 `.env` 和 `workflows` 放回原位

⑤ 重新运行 `install.bat` 安装新依赖

⑥ 运行 `start.bat` 启动

## 七、Docker 相关问题

### Docker 无法启动

① 在桌面右下角找到鲸鱼图标

② 右键 → **Restart Docker**

③ 如果仍然失败，重启电脑

### n8n 数据丢失

n8n 数据保存在 Docker 的数据卷中，删除容器不会丢失。

如需恢复：运行 `start.bat` 即可自动重建。

### 清理 Docker 占用空间

打开命令行（Win+R → 输入 `cmd`），执行：
```
docker system prune -a
```
输入 `y` 确认。这会清理不再使用的镜像和缓存，释放磁盘空间。

## 八、常用目录说明

| 目录 | 用途 | 可以删除 |
|------|------|----------|
| `logs\` | 运行日志 | 可以清理旧日志 |
| `storage\` | 历史记录数据库 | 不要删除 |
| `data\browser\` | 微信登录状态 | 登录失效时可删 |
| `n8n_data\` | n8n 工作流数据 | 不要删除 |

## 九、紧急联系清单

出现以下情况，请立刻联系技术支持：

- `.env` 文件中的 API Key 泄露（怀疑被盗用）
- 系统频繁崩溃无法使用
- 公众号账号出现异常登录提示
- 文章内容出现严重错误或违规内容
