# AI 公众号自动运营工作流 — 实施工作指南

> 按顺序执行，每一阶段完成后验证，再进下一阶段。

---

## 阶段〇：准备工作（先做完再开工）

### 0.1 确认你的服务器环境

- [ ] 一台能 24h 运行的机器（本地/云服务器），建议 4GB 内存以上
- [ ] 安装 Docker Desktop（[下载](https://www.docker.com/products/docker-desktop/)）
- [ ] 确认 Docker 能正常启动

### 0.2 确认公众号状态

- [ ] 服务号认证是否已完成？（未完成先不要进阶段二，发布 API 不可用）
- [ ] 登录 [公众号后台](https://mp.weixin.qq.com/) → 开发 → 基本配置：
  - [ ] 记录 AppID
  - [ ] 记录 AppSecret（如果已重置，用新的）
  - [ ] 把服务器 IP 加入白名单

### 0.3 准备品牌资料

- [ ] 品牌介绍（一段话级别的文字即可，先跑通再丰富）
- [ ] 2-3 篇你认为"写得好"的参考文章（风格模仿用）
- [ ] 确定你希望的文章风格：专业 / 亲切 / 犀利 / 幽默

## 阶段二：搭第一条最小链路（预计 1 天）

> 目标：手动输入一个主题 → AI 生成正文 → 创建公众号草稿。先不搞复杂功能，跑通最重要。

### 2.1 创建工作流文件

新建一个 n8n workflow，命名为 `公众号-核心链路-v1`。

---

### 2.2 节点 1：手动输入触发器

拖入 `Webhook` 节点，配置：
- HTTP Method: POST
- Path: `/article`
- Response: "已收到"

之后你可以用 curl 或 Postman 测试：
```bash
curl -X POST http://localhost:5678/webhook/article \
  -H "Content-Type: application/json" \
  -d '{"topic":"AI在制造业的应用"}'
```

---

### 2.3 节点 2：调用 AI 生成正文

拖入 `HTTP Request` 节点，连到 Webhook 后面。

配置：
- Method: POST
- URL: `https://api.deepseek.com/v1/chat/completions`
- Headers:
  ```
  Authorization: Bearer 你的DeepSeek-API-Key
  Content-Type: application/json
  ```
- Body (JSON):
  ```json
  {
    "model": "deepseek-chat",
    "messages": [
      {
        "role": "system",
        "content": "你是专业的公众号写手。根据用户提供的主题，写一篇结构完整的公众号文章。用 Markdown 格式输出，包含：标题、摘要、正文（含小标题分段）。语气专业、有深度。字数 1500 字左右。"
      },
      {
        "role": "user",
        "content": "请写一篇关于：{{ $json.body.topic }} 的公众号文章"
      }
    ],
    "temperature": 0.8,
    "max_tokens": 4096
  }
  ```

> 先用 DeepSeek 测试（便宜），后续再加模型切换。

提取结果：在 Response 里用 `body.choices[0].message.content` 取到文章 Markdown。

**验证**：手动触发一次，看能输出完整 Markdown 文章。

---

### 2.4 节点 3：Markdown 转公众号 HTML

拖入 `WeChat MD` 节点（社区节点），连到 AI 输出后面。

配置：
- 输入：上一步 AI 输出的 Markdown 文本
- 输出：WeChat HTML

**验证**：看输出内容是否包含 `<section>`、`<p>`、`style=` 等公众号风格的 HTML。

---

### 2.5 节点 4：创建草稿

拖入 `WeChat Official Account` 节点，连到格式化节点后面。

选择 Resource: Draft，Operation: Create。

填入：
- title: `{{ 从AI输出解析标题 }}`
- content: `{{ 格式化后的HTML }}`
- digest: `{{ 从AI输出解析摘要 }}`
- author: "你的公众号作者名"

> 提取标题/摘要可能需要加一个 `Function` 节点，用简单正则或分隔符从 Markdown 里拆。

**验证**：手动触发一次，去公众号后台"草稿箱"看到一篇草稿。

---

### 里程碑 1 🎯

**手动输入主题 → AI 生成文章 → 草稿出现在公众号后台。**

这条链路通了，后面的功能都是在这基础上加。

## 阶段三：加入安全检测（预计 0.5 天）

### 3.1 在格式化之后、创建草稿之前，插入敏感词检测

拖入 `HTTP Request` 节点，插入到格式化节点和公众号节点之间。

配置：
- Method: POST
- URL: `http://localhost:8080/wordscheck`
- Body (JSON):
  ```json
  {
    "content": "{{ 格式化后的文章纯文本 }}"
  }
  ```

### 3.2 加判断逻辑

在敏感词检测节点后面加一个 `IF` 节点：

条件：`$json.word_list.length === 0`

- 真 → 放行，进入创建草稿
- 假 → 走通知通道（先简单点：直接报错停止，后续再加分支）

**验证**：故意输入一个含敏感词的主题，看能拦截。

---

## 阶段四：发布链路（预计 1 天）

### 4.1 人工审核模式（先做这个）

在创建草稿成功后，加一个 `Webhook` 节点作为"审核确认入口"，再连到发布节点。

流程：
```
创建草稿 → 推通知（告诉你草稿ID）→ 你在公众号后台看草稿
→ 确认无误，curl 触发审核 Webhook → 自动发布
```

### 4.2 自动发布模式

在创建草稿后直接调用发布接口：

WeChat Official Account 节点：
- Resource: Draft
- Operation: Publish
- Media ID: `{{ 上一步返回的 draft_media_id }}`

### 4.3 发布结果处理

加 `IF` 判断发布结果：
- 成功 → 记录日志 + 推送成功通知
- 失败 → 推送错误通知（带上错误码和原因）

**验证**：走通"创建草稿 → 审核 → 发布成功"完整链路。

---

### 里程碑 2 🎯

**创建草稿 → 审核确认 → 成功发布到公众号。**

## 阶段五：加通知（预计 0.5 天）

### 5.1 选一个通知渠道

挑一个你团队在用的：

| 渠道 | 怎么接 |
|------|--------|
| 企业微信群机器人 | HTTP Request → Webhook URL |
| 飞书群机器人 | HTTP Request → Webhook URL |
| 钉钉群机器人 | HTTP Request → Webhook URL |
| 邮件 | n8n 内置 Email 节点 |

### 5.2 在关键节点后加通知

以下位置插入通知节点：

| 位置 | 内容 |
|------|------|
| 草稿创建成功 | "新文章草稿已生成：[标题]，请审核" |
| 发布成功 | "文章已发布：[标题]，链接：[URL]" |
| 检测到敏感词 | "⚠️ 文章含敏感词：[列表]，已拦截" |
| 发布失败 | "❌ 发布失败：[错误原因]" |

**验证**：发布一篇文章，群/邮箱收到通知。

## 阶段六：定时任务（预计 0.5 天）

### 6.1 把 Webhook 触发器换成 Schedule

拖入 `Schedule Trigger` 节点，替换原来的 Webhook。

配置：每天早上 9 点触发，只限工作日：
```
Cron: 0 9 * * 1-5
```

### 6.2 准备选题列表

如果每天生成不同主题，在 Schedule 后面加一个选题读取节点。

简单做法：弄一个数组，每天随机取一条：
```javascript
const topics = [
  "AI在制造业的5个落地场景",
  "2025年数字化转型趋势",
  "中小企业如何用好大模型"
];
return { topic: topics[Math.floor(Math.random() * topics.length)] };
```

**验证**：第二天早上 9 点自动跑，草稿箱出现新文章。

---

### 里程碑 3 🎯

**每天定时自动跑完整链路：选题 → 生成 → 检测 → 草稿 → 通知。**

## 阶段七：加功能模块（按需选做）

以下模块不互斥，做完一个验证一个。

### 7.1 知识库检索（0.5 天）

**做法**：在调用 AI 之前，先用 HTTP Request 查向量库，把结果拼到 Prompt 里。

简化版（先不搞向量库）：直接把品牌资料放在 n8n 变量里，拼进 System Prompt：

```
你是 XX 公司的公众号写手。公司背景：{品牌介绍}。
品牌风格：{语气描述}。
```

### 7.2 多模型切换（0.5 天）

在 AI 调用前加一个 `Switch` 节点：

```
选择模型 → 根据选择走不同的 HTTP Request 分支
        → OpenAI 分支：URL = api.openai.com
        → DeepSeek 分支：URL = api.deepseek.com
        → 通义千问分支：URL = dashscope.aliyuncs.com
```

每个分支的 Header 和 Body 格式略有不同，各自配置。

### 7.3 多模板（0.5 天）

在触发时让用户选模板类型，传到 AI Prompt 里：

```
如果是"营销推广"模板，使用以下结构：
1. 开篇钩子（80字）
2. 痛点分析（200字）
3. 产品方案（300字）
4. 客户案例（200字）
5. 行动号召（100字）
```

### 7.4 AI 配图（0.5 天）

**做法 1（免费）**：从 AI 输出中提取关键词 → 调 Unsplash API 搜图 → 拿到图片 URL 嵌入文章。

**做法 2（付费）**：调 DALL·E API 生图 → 下载图片 → 通过公众号节点上传为永久素材 → 获取 media_id 嵌入。

建议先用做法 1 跑通。

### 7.5 SEO 优化（0.5 天）

在正文生成完成后，再加一次 AI 调用，专门做 SEO：

Prompt：优化上文标题，包含核心关键词，摘要压缩到 120 字以内。

### 7.6 内容模仿（0.5 天）

流程：用户输入参考文章 → 加一个分析 Prompt（提取风格特征）→ 把风格注入生成 Prompt。

**注意**：参考内容用户手动粘贴，不自动爬取。

## 阶段八：错误处理加固（0.5 天）

### 8.1 添加 Error Trigger

在 n8n 工作流设置里，勾选 "Error Workflow"，错误时走一个专门的错误处理流。

### 8.2 关键节点的重试

在 `HTTP Request` 节点里设置 Retry：
- Max Retries: 3
- Retry Interval: 30s

### 8.3 发布结果轮询

微信公众号发布是异步的。发布接口返回后，可能过几分钟才看到结果。

加一个 `Wait` 节点等 5 分钟，再查一次发布状态。

## 实施原则

1. **先跑通，再优化**。阶段二的最小链路只花 1 天，不要上来就想做全。
2. **每步都验证**。别连做三步再测，做一个节点测一下。
3. **用最便宜的模型先跑**（DeepSeek），调通了再换贵的。
4. **人工审核模式先用起来**，稳定一周后再开自动发布。
5. **出了问题先看 n8n 的执行日志**，每个节点的输入输出都能看到。
