## 三、模块详细设计

### 3.1 触发层

```mermaid
flowchart LR
    subgraph 触发方式
        A["⏰ 定时触发<br/>Cron: 0 9 * * 1-5<br/>工作日早9点"]
        B["✋ 手动触发<br/>Webhook + 主题输入"]
        C["📋 批量队列<br/>Excel/Google Sheets<br/>选题列表逐条消费"]
    end

    subgraph 状态管理
        D["n8n 内置变量<br/>$json, $node, $workflow"]
        E["状态文件 JSON<br/>记录当前进度"]
    end

    A & B & C --> D --> E
```

| 触发方式 | 适用场景 | n8n 实现 |
|---------|---------|---------|
| Schedule Trigger | 每日/每周定时生成 | `Schedule Trigger` 节点 + Cron 表达式 |
| Webhook Trigger | 临时加急文章 | `Webhook` 节点接收 POST 请求 |
| Content Queue | 批量选题去重生成 | `Google Sheets` / `Spreadsheet File` 节点逐行读取 |

```mermaid
flowchart TB
```

**技术选型**：
- 向量库：`Qdrant`（本地部署）或 `ChromaDB`，都是 Docker 一键启动
- Embedding：用 n8n 的 HTTP Request 调 OpenAI `text-embedding-3-small`
- 检索策略：取 Top-5 片段，拼接后注入 Prompt 的 System Message

```mermaid
    RESULT -->|全部通过| PASS["✅ 放行 → 进入发布"]
    RESULT -->|有风险| FAIL["⚠️ 标记风险 → 推人工审核"]
```

**wordscheck 部署**：
```bash
# Docker 一键部署
docker run -d -p 8080:8080 --name wordscheck wordscheck-image

# n8n 中的 HTTP Request
curl -X POST http://localhost:8080/wordscheck \
  -H "Content-Type: application/json" \
  -d '{"content":"待检测的文章正文"}'

# 返回示例
{
  "code": "0",
  "msg": "检测成功",
  "return_str": "处理后的文本（敏感词已脱敏）",
  "word_list": [
    {"keyword": "艳情", "category": "色情", "position": "4-5", "level": "高"}
  ]
}
```

**判断逻辑**：

| 命中级别 | 处理方式 |
|---------|---------|
| 高 | 🛑 直接驳回，通知原因 |
| 中 | ⚠️ 自动脱敏后推送人工审核 |
| 低 | 📝 自动脱敏，附注风险说明，继续发布 |
| 无 | ✅ 直接放行 |

## 六、错误处理与异常恢复

```mermaid
flowchart TB
    subgraph ERRORS["错误分类"]
        E1["🔴 致命错误<br/>（工作流终止）"]
        E2["🟡 可恢复错误<br/>（重试后继续）"]
        E3["🟢 预警<br/>（继续但通知）"]
    end

    E1 --> CASE1["API密钥无效<br/>知识库不可用<br/>n8n节点崩溃"]
    E2 --> CASE2["网络超时<br/>token过期<br/>AI 返回格式异常"]
    E3 --> CASE3["文章字数不够<br/>SEO评分偏低<br/>配图缺失"]

    CASE1 --> ACT1["🛑 立即停止<br/>推紧急通知"]
    CASE2 --> ACT2["🔄 自动重试3次<br/>间隔30s/60s/120s<br/>仍失败则降级通知"]
    CASE3 --> ACT3["📝 记录日志<br/>附注风险继续<br/>不阻断流程"]
```

**微信发布错误码对照**：

| 微信错误码 | 含义 | 处理策略 |
|-----------|------|---------|
| 40001 | access_token 无效 | 自动刷新 token 重试 |
| 40007 | 媒体文件格式错误 | 检查图片格式，转 JPG/PNG |
| 40125 | AppSecret 错误 | 致命错误，通知管理员 |
| 45009 | API 调用频率超限 | 等待 5 分钟后重试 |
| 48001 | 无 API 权限 | 确认服务号已认证 |
| 85002 | 审核中，不能重复提交 | 跳过，等待上次结果 |
| 88001 | 内容含有敏感信息 | 标记为"需人工审核" |

```mermaid
flowchart LR
```

> **设计原则**：
> - **不自动爬取**：所有参考内容由用户手动输入
> - **不原文照搬**：AI 只提取风格特征，内容完全重新生成
> - **不存储原文**：参考内容用完即弃，不落盘

## 九、完整数据流（端到端）

```mermaid
flowchart TB
    START([开始]) --> TRIGGER{"触发方式"}

    TRIGGER -->|定时| INPUT_TIME["⏰ Cron触发<br/>加载预设选题"]
    TRIGGER -->|手动| INPUT_MANUAL["✋ Webhook触发<br/>用户输入主题"]

    INPUT_TIME & INPUT_MANUAL --> KB["📚 检索知识库<br/>Top-5相关片段"]

    KB --> PROMPT["🔧 组装 Prompt<br/>系统指令 + 模板 + 知识库"]

    PROMPT --> AI_STEP1["Step 1: 行业分析"]
    AI_STEP1 --> AI_STEP2["Step 2: 用户分析"]
    AI_STEP2 --> AI_STEP3["Step 3: 结构规划"]
    AI_STEP3 --> AI_STEP4["Step 4: 正文生成"]
    AI_STEP4 --> AI_STEP5["Step 5: SEO优化"]

    AI_STEP5 --> QUALITY["✅ AI 质量检查"]

    QUALITY --> CHECK_PASS{"质量通过？"}
    CHECK_PASS -->|否| FIX["🔄 返回修改"]
    FIX --> AI_STEP4

    CHECK_PASS -->|是| WC["🚫 wordscheck<br/>敏感词扫描"]

    WC --> WC_PASS{"通过？"}
    WC_PASS -->|否| ALERT["⚠️ 风险预警<br/>推人工审核"]
    WC_PASS -->|是| IMG["🖼️ 配图匹配"]

    IMG --> FMT["📐 格式化 HTML"]

    FMT --> DRAFT_API["📝 创建草稿<br/>公众号 API"]

    DRAFT_API --> DRAFT_OK{"成功？"}
    DRAFT_OK -->|否| ERR_HANDLE["🔄 错误处理+重试"]
    ERR_HANDLE --> DRAFT_API

    DRAFT_OK -->|是| MODE_SWITCH{"发布模式"}

    MODE_SWITCH -->|人工| NOTIFY_REVIEW["📬 推审核通知"]
    NOTIFY_REVIEW --> WAIT_APPROVAL["⏳ 等待审核"]
    WAIT_APPROVAL --> APPROVED{"审核结果"}
    APPROVED -->|通过| PUB_FINAL["🚀 发布"]
    APPROVED -->|驳回| REVISE["📝 退回修改"]

    MODE_SWITCH -->|自动| PUB_FINAL

    PUB_FINAL --> PUB_OK{"发布成功？"}
    PUB_OK -->|是| LOG_SUCCESS["✅ 记录日志<br/>推送成功通知"]
    PUB_OK -->|否| LOG_FAIL["❌ 记录失败<br/>推送异常通知"]

    LOG_SUCCESS & LOG_FAIL --> END([结束])
```
